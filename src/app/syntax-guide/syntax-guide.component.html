Sides

You can search based on the sides of a tile by using the sides: or s: keywords.

These keywords should be followed by 1-4 letters representing the sides of the tile in clockwise order.

Known sides are:

  • c for city
  • f for field
  • r for road
  • s for river (think "stream")

    • Orientation does not matter for the purpose of this search. Any rotation of the search parameters will be considered an acceptable result (e.g. s:sfrc, s:csfr, s:rcsf, and s:frcs are all equivalent).

    If you want to search for a tile with city tiles on opposite sides and don't care about the sides in between, you should use the wildcard character - to represent the sides you don't care about.

    s:ccff

    Tiles that have two adjacent castles, then two fields

    s:r-r-

    Tiles that have roads on opposite edges, and anything on the other two

    sides:s

    Tiles where at least one of the edges is a river

    Tile Contents

    You can search tiles based on whether they containa certain game feature by specifying the feature name as a search keyword.

    Known features are:

  • Monasteries (written as monastery, mon, or m)
  • Coat of Arms (written as coa, shield, or coat-of-arms)
  • Gardens (written as garden or g)
  • Inns (written as inn or i)
  • Cathedrals (written as cathedral, cath, or c)

    • These searches can be performed either inclusively or exclusively. To perform an exclusive search, the search term should be followed with a colon and an operand meaning false (such as f). To perform an exclusive search, the search term should be followed with a colon and an operand meaning true (such as t).

    m:t

    Tiles with monasteries

    coa:f

    Tiles without coat of arms

    garden inn:f

    Tiles with a garden, but without an inn

    Expansions

    You can search for tiles from a specific expansion by using the expansion: or exp: keywords, followed by the name of the expansion to search for.

    Valid forms for the expansion name are: camelCase, snake_case, and kebab-case, as well as abbreviations wherever one can be inferred. For the purposes of this search, capitalization does not matter.

    expansion:inns-and-cathedrals

    Tiles from the Inns and Cathedrals expansion

    exp:baseset

    Tiles from the Traders and Builders expansion

    Printings

    You can search for tiles from a specific printing by using the printing: or print: keywords, followed by the name of the printing to search for.

    Valid forms for the printing name are: camelCase, snake_case, and kebab-case, as well as abbreviations wherever one can be inferred. For the purposes of this search, capitalization does not matter.

    Boolean Expressions

    All of the above search terms work by searching the list of all existing tiles to see if they match a given criteria. If you want to search for tiles that meet multiple different criteria, that contain one or more different features, or the inverse of a search term, you can use boolean expressions.

    By default, all search terms are combined with the logical 'and' operator. This means all tiles found by the search must match all criteria specified. If preferred, you can use the keyword and (or &) to explicitly show this relationship in a search.

    If you want to search for tiles that meet at least one of two search terms, you would use the logical 'or' operator (written as either or or |). This must be explicitly stated in the search, as otherwise the default 'and' operator will be used to join two search functions.

    If you want to search for all tiles that do not meet a certain criteria, you can use the logical 'not' operator (specified by either not or !). When placed before a search keyword, this will return the inverse of what that keyword is searching for.
